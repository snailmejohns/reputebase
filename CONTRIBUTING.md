# Contributing to ReputeBase

Thank you for your interest in contributing to ReputeBase! This document provides guidelines and instructions for contributing.

## Getting Started

1. Fork the repository
2. Clone your fork: `git clone https://github.com/yourusername/reputebase.git`
3. Create a branch: `git checkout -b feature/your-feature-name`
4. Make your changes
5. Commit your changes: `git commit -m 'Add some feature'`
6. Push to the branch: `git push origin feature/your-feature-name`
7. Open a Pull Request

## Development Setup

### Install Dependencies

```bash
npm run install:all
```

### Run Tests

```bash
npm run test:contracts
```

### Build Contracts

```bash
npm run build:contracts
```

## Code Style

- Follow Solidity style guide for smart contracts
- Use ESLint for JavaScript/TypeScript code
- Write clear commit messages
- Add comments for complex logic

## Pull Request Process

1. Update documentation if needed
2. Add tests for new features
3. Ensure all tests pass
4. Update CHANGELOG.md if applicable
5. Request review from maintainers

## Reporting Issues

Use GitHub Issues to report bugs or suggest features. Include:
- Description of the issue
- Steps to reproduce
- Expected vs actual behavior
- Environment details

## Questions?

Join our Discord or open a Discussion on GitHub.
